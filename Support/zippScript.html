zippScript_Version( {Format} )

 

Description

Returns the version number of the currently installed plug-in.

 

Parameters

Format - an optional parameter specifying an alternate format of version information.

 

Examples

zippScript_Version returns 2.2

zippScript_Version( ““ ) returns 2.2

zippScript_Version( “short” ) returns 2.2

zippScript_Version( “long” ) returns zippScript 2.2

zippScript_Version( “platform” ) returns Mac OS X (Intel) when FileMaker 8.5 is running on an Intel Mac.

 

Note

If the plug-in is not installed, calling this function will return an invalid result. If the plug-in is installed but disabled, the function will return an empty string. This feature may be used to test for the existence of the plug-in if your solution requires it to enable some functionality.

 

zippScript_PerformScript( FileName; ScriptName {; Param; Control} )

 

Description

Allow a script to be run from any FileMaker calculation dialog.

 

Parameters

FileName - the name of the open file containing the script to be run. This will typically be set to Get(FileName), but may be a text string explictly naming the file (“MyFile.fp7”), a text field containing the name of the file, or a calculation or custom function returning the name of the file.

ScriptName - the name of the script to run. As with the FileName parameter, the name may be supplied by text string, field or calculation.

Param - an optional parameter containing a string to be passed to the specified script as a text script parameter. Multiple script parameters may be passed to the script using any delimiter you choose. The default value is an empty script parameter.

Control - an optional parameter used to modify FileMakerÕs handling of other running scripts when the specified script is run. This value may be one of “halt”, “exit”, “resume”, “pause”, or the corresponding numerical values 0, 1, 2 or 3. The default value is “pause”.

 

Note

If script triggers have been disabled by a previous use of the zippScript_DisableTriggers function, the call to this function will be ignored and no script will be triggered.

An error code may be returned if the named file is not open or the named script cannot be found in that file.

 

zippScript_ScheduleScript( Time; FileName; ScriptName {; Param; Control} )

 

Description

Schedule a single script trigger to be performed at some future time.

 

Parameters

Time - a Date, Time or TimeStamp specifying the time at which the scheduled script is to be triggered.

Passing a Date will schedule a trigger just past midnight on the morning of the given Date. Passing a Time will schedule a trigger at the given time on todayÕs date. Passing a TimeStamp will schedule a trigger at that Date and Time.

FileName - the name of the open file containing the script to be run. This will typically be set to Get(FileName).

ScriptName - the name of the script to run.

Param - an optional parameter containing a string to be passed to the specified script as a text script parameter. Multiple script parameters may be passed to the script using any delimiter you choose. The default value is an empty script parameter.

Control - an optional parameter used to modify FileMakerÕs handling of other running scripts when the specified script is run. This value may be one of “halt”, “exit”, “resume”, “pause”, or the corresponding numerical values 0, 1, 2 or 3. The default value is “pause”.

 

Examples

zippScript_ScheduleScript( “1/1/2007”; “MyFile”; “NewYearScript” ) will perform the script just as the clock strikes on New YearÕs day.

zippScript_ScheduleScript( Get(CurrentTimeStamp) + 30; “MyFile”; “MyScript”; SomeField ) will perform the script in 30 seconds from now, passing the contents of SomeField as a script parameter.

zippScript_ScheduleScript( Get(CurrentHostTimeStamp) + 60; “MyFile”; “MyScript” ) will schedule the script relative to the serverÕs clock, which may be a different time than the local clock.

 

zippScript_GetScheduledScript

 

Description

Returns information about the scheduled script trigger in a pipe-delimited (“|”) string.

 

Examples

zippScript_GetScheduledScript returns an empty string when no script trigger has been scheduled, or after a scheduled script trigger has been activated.

zippScript_GetScheduledScript returns “12/31/2006 11:59:59 PM|zippScript Example|ShowMessage|Happy New Year!” when a script trigger has been scheduled to run a script to display a message just before Midnight on New YearÕs Eve.

 

Note

This function will return an empty string when no script trigger has been scheduled, or after a scheduled script trigger has been activated.

Scheduled script triggers may not be activated exactly at the specified time if FileMaker is busy or if script triggers have been disabled.

FileMakerÕs text manipulation functions may be used to extract individual pieces from the string of scheduled script trigger information.

 

zippScript_CancelScheduledScript

 

Description

Cancel a previously scheduled script trigger before the event time has occurred.

 

Examples

zippScript_CancelScheduledScript cancels any pending scheduled script trigger. No value is returned.

 

Note

This function may be called at any time, even if a script has not been scheduled or the event time has occurred and the scheduled script has already been triggered.

Any pending scheduled script trigger will be canceled and the details of that trigger will be forgotten. A subsequent call to zippScript_GetScheduledScript will return an empty string.

You may want to call this function after calling zippScript_DisableTriggers to ensure a scheduled script trigger whose event time occurs during the period that triggers are disabled does not attempt to run as soon as events are re-enabled.

You may also want to call this function if you close the file but do not quit FileMaker. A scheduled script trigger calling a script in the closed file will attempt to perform the script at the scheduled time and will silently fail when the file cannot be found.

 

zippScript_IdleScript( Minutes; FileName; ScriptName {; Param; Control} )

 

Description

Schedule a single script trigger to be performed after a period of no user activity.

 

Parameters

Minutes - a number specifying the minutes of no user activity that must pass before the idle script is to be triggered.

FileName - the name of the open file containing the script to be run. This will typically be set to Get(FileName).

ScriptName - the name of the script to run.

Param - an optional parameter containing a string to be passed to the specified script as a text script parameter. Multiple script parameters may be passed to the script using any delimiter you choose. The default value is an empty script parameter.

Control - an optional parameter used to modify FileMakerÕs handling of other running scripts when the specified script is run. This value may be one of “halt”, “exit”, “resume”, “pause”, or the corresponding numerical values 0, 1, 2 or 3. The default value is “pause”.

 

Examples

zippScript_IdleScript( 30; “MyFile”; “LogoutUser” ) will perform the LogoutUser script after approximately 30 minutes of no user activity.

 

zippScript_GetIdleScript

 

Description

Returns information about the idle script trigger in a pipe-delimited (“|”) string.

 

Examples

zippScript_GetIdleScript returns an empty string when no idle script trigger has been set, or after an idle script trigger has been activated.

zippScript_GetIdleScript returns “30|SomeFile|LogoutUser” when an idle script trigger has been set to run the LogoutUser script after 30 minutes of no user activity.

 

Note

This function will return an empty string when no idle script trigger has been set, or after an idle script trigger has been activated.

FileMakerÕs text manipulation functions may be used to extract individual pieces from the string of scheduled script trigger information.

 

zippScript_CancelIdleScript

 

Description

Cancel a previously set idle script trigger before the time-out event has occurred.

 

Examples

zippScript_CancelIdleScript cancels any pending idle script trigger. No value is returned.

 

Note

This function may be called at any time, even if an idle script has not been scheduled or the time-out event has occurred and the idle script has already been triggered.

Any pending idle script trigger will be canceled and the details of that trigger will be forgotten. A subsequent call to zippScript_GetIdleScript will return an empty string.

You may want to call this function if you close the file but do not quit FileMaker. An idle script trigger calling a script in the closed file will attempt to perform the script and will silently fail when the file cannot be found.

 

zippScript_EnableTriggers

 

Description

Allow zippScript to perform immediate and scheduled script triggers.

 

Examples

zippScript_EnableTriggers will allow processing of script triggers. No value is returned.

 

Note

This function will re-enable script triggers for all open files.

A pending scheduled script trigger whose event time has passed during the period that script triggers were disabled will be activated shortly after script triggers are re-enabled.

Any pending scheduled script trigger may be canceled by calling zippScript_CancelScheduledScript after calling zippScript_DisableTriggers or before calling this function.

 

See also zippScript_DisableTriggers.

 

zippScript_DisableTriggers

 

Description

Prevent zippScript from performing any script triggers.

 

Examples

zippScript_DisableTriggers will prevent processing of all script triggers until triggers are re-enabled. No value is returned.

 

Note

This function will disable script triggers for all open files.

You will often want to do this as part of a scripted process that modifies fields that have script triggers attached so that the changes made to those records will not attempt to run the script triggers. You may want to disable triggers at the beginning of the script and re-enable them at the end of the script.

While triggers are disabled, any call to zippScript_PerformScript will be ignored and no script will be triggered. You may still use zippScript_ScheduleScript, but no scheduled script triggers will be activated when event times occur.

A pending scheduled script trigger will not be automatically canceled but will be activated when script triggers are re-enabled.

 

See also zippScript_EnableTriggers.

 

zippScript_GetTriggerState

 

Description

Returns a boolean value indicating whether script triggers are currently enabled.

 

Examples

zippScript_GetTriggerState returns 1 if script triggers are currently enabled or 0 if script triggers have been disabled.

 

Note

Script triggers may be temporarily enabled or disabled by using the zippScript_EnableTriggers or zippScript_DisableTriggers functions, respectively.

 

See also zippScript_EnableTriggers and zippScript_DisableTriggers.
